## What `/bugfix` Discovered

Investigating why `interactive.router` is not in `api/main.py:295-359` revealed not one but a chain of missing modules:

1. **`api/exceptions.py` was missing.** Six files import from it (`api.routers.interactive`, `api.services.{finding_investigator, false_positive_analyzer, remediation_generator, model_router, session_manager, bulk_analyzer}`). None were importable. **Fixed in this commit** — created `api/exceptions.py` re-exporting `InsufficientCreditsError` + `CreditTransactionError` from `api.services.credit_service`, plus a new `UnauthorizedError` class.

2. **`api/services/claude_service.py` is also missing.** Two files reference it:
   - `api/services/bulk_analyzer.py:17`: `from ..services.claude_service import claude_service`
   - `api/services/feedback_processor.py:22, 33`: `from api.services.claude_service import ClaudeService`

   `bulk_analyzer.py` is imported by `interactive.py:55` (the bulk-analyze endpoints at `/v1/interactive/bulk-*`). Without the `claude_service` module, importing `bulk_analyzer` raises `ModuleNotFoundError`, which cascades up: `interactive.router` cannot be loaded.

3. **No fallback path that doesn't go through bulk_analyzer.** Cannot register `interactive.router` partially — Python's module loader fails at the first missing import.

## Why `/bugfix` Will Not Auto-Fix This

`claude_service.py` is the missing implementation of a Claude/Anthropic LLM API wrapper. `bulk_analyzer.py:147` calls `await claude_service.analyze_with_claude(prompt, ...)`. The signature does not match the existing `LLMService` (`api/services/llm_service.py:39`), which uses `analyze_threat(LLMAnalysisRequest)`. So a re-export shim won't work — building `claude_service` is feature work.

Per the bugfix skill rules:
- "Don't add features beyond what was asked"
- "Always run the full test suite after the fix"
- Three-strike rule: if I can't get a clean fix in 3 attempts, escalate

Two missing modules (`api.exceptions`, `claude_service`) plus a non-trivial signature mismatch puts F1.7 firmly in feature-work territory. `/bugfix` ends here.

## Remediation Path (when someone takes this on)

1. **Decide the LLM service architecture.** Either:
   - **(a) Make `claude_service` a thin shim over `llm_service`** — adds an `analyze_with_claude(prompt, ...)` method that maps to `LLMService.analyze_threat(LLMAnalysisRequest)`. Need to map the bulk-analyzer's prompt-shaped calls to the request-shaped LLM service. Touches both `bulk_analyzer.py` (call sites) and the new `claude_service.py`.
   - **(b) Refactor `bulk_analyzer` to use `llm_service` directly.** Rip out the `claude_service` indirection. Smaller surface but rewrites the bulk path.
   - **(c) Drop bulk-analyze endpoints from interactive.py.** Comment out the `from api.services.bulk_analyzer import (...)` line and the `/bulk-*` routes that use it. Keeps the other 30+ interactive routes shippable. Lose the bulk feature.

2. **Run `pytest api/tests/test_interactive_router_registered.py`** with the `@pytest.mark.skip` removed — both tests should pass.

3. **Verify in production:** `curl -sS https://api.sigilsec.ai/v1/interactive/investigate -X POST` should return 401 (or 422 on validation), never 404.

4. **Check Stripe Pro feature contract.** The pricing page advertises:

# Evidence: US-109 — Modernize remediation generator + attack-chain tracer

**Date:** 2026-06-11 · **Feature:** F-009 · **Branch:** feature/sigil-pro-fable

## Change
- **`api/services/remediation_generator.py`**: pinned `"claude-3-sonnet-20240229"` → `llm_config.model` (standard model for code generation). `_call_llm_service` now passes `model=model` on the request and uses `LLMAnalysisType.BEHAVIORAL_PATTERN` — the previous `VULNERABILITY_ANALYSIS` is not an enum member (same latent AttributeError class as US-108: the remediation path crashed before reaching the LLM).
- **`api/services/attack_chain_tracer.py`**: `model_used` for credit deduction and the analysis request both use `llm_config.deep_model` (claude-fable-5 — multi-step attack-chain reasoning is the deep-model use case per PRD). `custom_prompt` moved from post-construction attribute assignment — which Pydantic v2 rejects with ValueError for undefined fields, so tracing crashed and every call returned the fallback chain — into a real constructor field.
- **`api/llm_models.py`** (coupled, additive): `LLMAnalysisRequest.custom_prompt: str | None` — verbatim prompt override.
- **`api/services/llm_service.py`** (coupled, additive): `_build_analysis_prompt` returns `custom_prompt` when set; `_perform_analysis` reports `model_used=request.model or llm_config.model` (previously always reported the config default even when overridden).

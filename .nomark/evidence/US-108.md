## Change
- **`api/services/finding_investigator.py`**:
  - `_get_model_and_cost` re-keyed from retired IDs (claude-3-haiku/sonnet/opus-2024xx) to config: quick‚Üí`llm_config.fast_model`, thorough‚Üí`llm_config.model`, exhaustive‚Üí`llm_config.deep_model`. Costs now read the canonical `SCAN_COSTS` per-depth keys (4/8/16) ‚Äî the old `SCAN_COSTS["investigate_finding"]` key **does not exist**; every investigation depth crashed with KeyError (latent bug #1, pre-existing, untested).
  - `_call_llm_for_investigation` rewritten: model override now travels with the request (`LLMAnalysisRequest.model`) instead of temporarily mutating `llm_service.llm_config.model` ‚Äî that hack raced concurrent investigations and referenced `llm_service.llm_config`, an attribute that doesn't exist (`hasattr` guard silently skipped the override, so the depth model was never actually applied ‚Äî latent bug #2). Also fixed `LLMAnalysisType.VULNERABILITY_ANALYSIS` ‚Üí `BEHAVIORAL_PATTERN`; the former is not a member of the enum (latent bug #3 ‚Äî AttributeError on every call). max_tokens: 4000 for deep model, else 2000 (was `"opus" in model`, which would now misfire on claude-opus-4-8 mid-tier).
- **`api/llm_models.py`** (coupled, additive): `LLMAnalysisRequest.model: str | None = None` ‚Äî per-request model override.
- **`api/services/llm_service.py`** (coupled, additive): `_perform_analysis` passes `request.model` to `call_llm_api` (US-103 param); docstring example ID updated.
- **`api/routers/interactive.py`** (coupled consumer, mechanical per pre-authorization): `CreateSessionRequest.model_preference` default `"claude-3-haiku"` ‚Üí `default_factory: llm_config.fast_model`; investigate docstring credit/model text aligned to actual SCAN_COSTS (exhaustive is 16, not 24 ‚Äî the docstring was wrong even before).
- **`api/services/explanations.py`**: no change needed ‚Äî it is a static rule-explanation registry with no LLM calls or model IDs (verified by test).

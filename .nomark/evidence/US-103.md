**Date:** 2026-06-11 · **Feature:** F-009 · **Branch:** feature/sigil-pro-fable

## Change (`api/services/llm_service.py`)
- New `LLMRefusalError(model, category)` — typed, terminal.
- `call_llm_api` anthropic branch checks `stop_reason == "refusal"` **before** reading `content[0]`; pre-output (empty content) and mid-stream (partial content) refusals are both handled — partials are discarded, never returned.
- Refusal on `llm_config.deep_model` (claude-fable-5) → exactly one retry on `llm_config.model` (claude-opus-4-8). Refusal anywhere else (incl. the fallback) → `LLMRefusalError` with the `stop_details.category`.
- tenacity: `retry_if_not_exception_type(LLMRefusalError)` — refusals are excluded from transient-error retries (identical content re-sent to the same classifier cannot succeed; prevents 3× wasted spend).
- Anthropic payloads no longer send `temperature` (Fable 5 / Opus 4.8 reject sampling params) and never send a `thinking` config (Fable 5 rejects explicit config; omission = adaptive default).

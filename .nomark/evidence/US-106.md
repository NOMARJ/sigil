# Evidence: US-106 — FP adjudication service (Fable 5, structured verdict)

**Date:** 2026-06-11 · **Feature:** F-009 · **Branch:** feature/sigil-pro-fable

## Change
- **New `api/services/fp_adjudicator.py`**: `FPAdjudicator.adjudicate(finding, code_context)` → validated `{classification: benign_dual_use|suspicious|malicious, confidence: 0..1, rationale}`. Prompt carries rule id, pack provenance, location, snippet, and code context bounded to 8K chars (`[context truncated]` marker). Uses `llm_config.deep_model` (claude-fable-5) through the US-103 refusal-fallback path; verdicts requested as API-enforced `output_config` json_schema (additionalProperties: false, enum-locked classification) — no free-text parsing. `AdjudicationError` on any contract violation (bad JSON, unknown classification, confidence out of [0,1], missing rationale). Refusals on both models propagate as `LLMRefusalError` — a refusal is not a verdict.
- **`api/services/llm_service.py`** (additive, coupled necessity — itemized per owner's pre-authorization pattern): optional `output_config` param on `call_llm_api`, included in the Anthropic payload when present and preserved across the refusal fallback retry.

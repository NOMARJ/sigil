## Codebase Patterns

- API services use credit_service for all LLM operations
- Frontend components fetch credit balance on mount
- Use Claude Haiku for simple tasks, Sonnet for complex analysis
- All interactive features deduct credits atomically via stored procedures
- LLM prompts are built using template functions for consistency
- Git operations use subprocess with timeout for safety
- Dashboard pages use Next.js with TypeScript

  - api/models/suppression_rules.py
  - api/services/feedback_processor.py
  - dashboard/components/FeedbackButtons.tsx
  - api/routers/interactive.py (added /feedback endpoints)
  - api/tests/test_feedback_learning.py
- **Learnings:**
  - Feedback processing creates suppression rules when confidence >= 0.8
  - False positives with 95% confidence trigger complete suppression
  - Test file patterns (*/tests/*, */spec/*) are recognized and grouped
  - Team suppression rules require share_with_team flag and team_id
  - Personal rules only affect user's own scans by default
  - Confidence adjustments range from -1.0 to +1.0 based on feedback
  - Contradictory feedback weakens rules (effectiveness score)
  - Rules deactivate when effectiveness drops below 0.3
  - Marking test file pattern as safe successfully suppresses similar test files
